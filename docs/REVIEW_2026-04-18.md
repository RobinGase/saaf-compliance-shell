# Second independent review — 2026-04-18

Tracking doc for the second code review of saaf-compliance-shell. Items
are grouped by severity; the order within each group reflects the order
a v0.8.6 patch sweep should take them in.

Each item links to the identifier used in the review body so a
reader can match up. Items that span multiple files carry a short
"touches" note.

## Verdict summary

Reviewer called out three structural strengths:

1. Honest scope statement — docs and README both say what the shell is
   *not*.
2. Defence-in-depth visible in code (`output_scan.py` bypass-path
   re-scan of the twelve rails, with audit events for every fire).
3. Audit log quality — fcntl cross-process locking, crash-truncation
   healing, canonical-JSON hashing, genesis record, verifier. Called
   the "cleanest part of the codebase".

Rough quality gradient per the reviewer:
- Production-grade: `modules/audit/*`, `modules/isolation/network.py`.
- Careful regex work: `modules/guardrails/*_rule.py`.
- Most open issues: `modules/guardrails/service.py`, `modules/isolation/runtime.py`.

## Critical — must land in v0.8.6

- [x] **C1** `pyproject.toml` version is 0.7.1 while the tag is v0.8.5. Either
      bump on release or drive from `setuptools_scm`. Touches
      `pyproject.toml`. **Done:** hardcoded 0.8.5 with a release-script
      cross-check against `git describe`; setuptools_scm migration
      deferred to v0.9. (batch 1)
- [x] **C2+C3** `modules/isolation/runtime.py` emits `session_start`
      *before* VM setup and only `vm_exit` on the happy path. Failure
      modes (iptables setup, NFS start, Firecracker non-zero exit)
      leave the chain reading "start → end, nothing in between". Add
      explicit `session_setup_failed`, `nfs_start_failed`,
      `vm_nonzero_exit` events; drop `check=True` on
      `launch_firecracker` and record the real `returncode`.
      **Done:** `run_manifest` emits `vm_exit` with status=failed
      and `reason`/`exception_class`; `_run_commands` now inspects
      `returncode` explicitly and emits a `command_failed` audit
      event with captured stderr before raising. (batch 4)
- [x] **C4** `launch_firecracker` uses `subprocess.run(...,
      capture_output=True)`. Buffers the entire guest console in RAM
      until exit; a long/hung session leaves no live console log to
      debug. Stream to file: `stdout=open(console_log_path, "wb")`,
      `stderr=subprocess.STDOUT`. Touches
      `modules/isolation/runtime.py`. **Done:** streams stdout to
      `<console>.log` and stderr to `<console>.log.stderr`; non-zero
      exits raise `CalledProcessError` with the stderr tail. (batch 4)
- [x] **C5** `_neutral_cwd_for_colang_imports()` is a process-wide
      `os.chdir` under a context manager — races on any concurrent
      thread that reads CWD. Interim: `threading.Lock` + document
      single-worker. Long-term (v0.9): rename `guardrails/` →
      `guardrails_config/` so the CWD-shadow stops being reachable.
      Touches `modules/guardrails/service.py`. **Done:** chdir wrapped
      in `_CWD_CHDIR_LOCK`, `workers=1` made explicit in
      `start-guardrails-local.py`; rename deferred to v0.9. (batch 2)
- [x] **C6** `INJECTION_PATTERNS` is a lowercase substring match.
      Trivially bypassable (`"ignore  all"` with double space,
      Cyrillic homoglyphs, paraphrase). Correct per reviewer: the
      LLM `self_check_input` is the real gate, but the code doesn't
      say so. Add a module docstring stating "this is a tripwire,
      not a filter"; audit-log every hit. Long-term merge into a
      normalised-token scan. Touches
      `modules/guardrails/service.py`. **Done:** renamed to
      `DEFAULT_*`, added tripwire docstring listing bypass classes,
      audit events now carry category + matched pattern. (batch 1)
- [x] **C7** `_recover_quoted_llm_value` fishes LLM output out of a
      nemoguardrails exception string. Fragile: nemoguardrails owns
      that format. Emit an audit event for every attempt — success
      and failure — naming the exception class. Touches
      `modules/guardrails/service.py`. **Done:**
      `guardrails_salvage_attempt` event captures
      `exception_class` + `salvaged` bool on every attempt. (batch 2)

## High — on the near-term roadmap

- [ ] **H1** `_encode_boot_value` hand-rolled escape only handles
      space. A manifest entrypoint containing `"`, `'`, `\`, `$`, or
      a newline silently malforms kernel args or injects a second
      parameter. Validate in `_check_agent` at manifest-validation
      time (regex `^[A-Za-z0-9_./:-]+$` or similar; reject other
      chars). Touches `modules/isolation/vm_config.py`,
      `modules/validation/manifest.py`.
- [ ] **H2** `start_nfs_server` sends stdout/stderr to `DEVNULL`.
      When NFS won't start, the operator sees the downstream
      Firecracker boot failure, not the NFS error. Route to a
      session-scoped log file, same shape as the Firecracker console
      log. Touches `modules/isolation/nfs.py`.
- [ ] **H3** `DEFAULT_NFS_PORT = 11111` is static. Two concurrent
      sessions collide. v1 scope is single-session; enforce with a
      host-wide lockfile and document, or make the port per-session
      and insert into the `INPUT` iptables rule the same way.
- [x] **H4** `OFF_TOPIC_PATTERNS` includes `"recipe"` (false-positives
      on "recipe for disaster", risk-register recipes) and
      `"stroopwafels"` (a sentinel that escaped into prod). Move both
      pattern lists to `guardrails/config.yml` so they can be
      reviewed/tuned without a code change; audit-log every hit.
      Touches `modules/guardrails/service.py`,
      `guardrails/config.yml`. **Done:** both pattern lists now live
      in `config.yml` under `preflight_injection_patterns` and
      `preflight_off_topic_patterns`; `recipe` and `stroopwafels`
      dropped. (batch 1)
- [x] **H5** `verify_log` uses `record.pop("event_hash")` + restore.
      An exception in `_canonical_json` between pop and restore would
      leave the record un-restored. Build a throwaway dict instead:
      `to_hash = {k: v for k, v in record.items() if k !=
      "event_hash"}`. Touches `modules/audit/log.py`. **Done.** (batch 1)
- [ ] **H6** `_count_session_events` re-scans the entire audit log on
      every `end_session` — O(N) per close, painful once retention
      grows. Read backwards from EOF and stop at the current
      session's `session_start`. Lower priority; post-v1. Touches
      `modules/audit/log.py`.
- [x] **H7** `_build_rails` is `lru_cache`d on `(config_path,
      model_name, self_check_url)` — never invalidated on disk
      change. Redeploy the rail config → service keeps the old
      copy. Include `config.yml` mtime in the cache key, or probe
      mtime in `get_rails` and rebuild on change. Document in the
      runbook. Touches `modules/guardrails/service.py`. **Partial:**
      preflight patterns now self-invalidate on mtime change
      (`_load_preflight_patterns_cached` keyed on `st_mtime_ns`).
      `_build_rails` itself still caches without mtime — the
      LLMRails rebuild is heavy and blast-radius is different
      (requires a full reload anyway). Tracked for v0.9. (batch 3)
- [ ] **H8** `privacy_router` uses `httpx.AsyncClient` per request —
      fresh TCP + TLS handshake each inference. Hold a module-level
      client, reuse via a lifespan hook. Touches
      `modules/router/privacy_router.py`.
- [x] **H9** `/health` returns `ok` even if `AUDIT_LOG_PATH` is on a
      full or read-only volume. Next write fails silently via
      `logger.warning`. Probe writability (`os.access` + attempt a
      zero-byte `open(path, "a")`) and return 503 if it fails. Touches
      `modules/guardrails/service.py`,
      `modules/router/privacy_router.py`. **Done** for the guardrails
      service (`mkdir -p` parent + open-append probe → 503 on
      OSError; new test covers both paths); privacy_router /health
      still to do in batch 5. (batch 3)
- [x] **H10** `_log_route_decision` catches `OSError` only. Some
      filesystems raise non-`OSError` from `f.truncate()` or
      `fcntl.flock`. Silently dropping an audit event is the failure
      an auditor cares about most. Widen the except, escalate to
      `logger.error`. Touches
      `modules/router/privacy_router.py`. **Done:** widened to
      bare `Exception` with `exc_info=True`. (batch 3)

## Pushback the maintainer noted on first pass

- **C5 scope** — the `chdir` race window is narrower than the review
  framed it: `_build_rails` is `lru_cache`-guarded, so `chdir` only
  fires on cold cache-miss. Under uvicorn's single-worker async loop
  + lru_cache's internal serialisation, the race is cold-start +
  worker-count > 1. A `threading.Lock` closes it for the interim; the
  clean fix is renaming `guardrails/` → `guardrails_config/` in v0.9.
- **Review state** — the reviewer read pre-`ac6ac74` code, so they
  did not see the happy-path Colang wiring just shipped in v0.8.5.
  The bypass scanner praise still stands; it is now genuine
  defence-in-depth rather than the primary control.
